# 오븐로드 프로젝트 구조 설명

팀원들이 코드를 이해할 수 있도록 전체 구조를 정리한 문서입니다.

---

## 1. 전체 그림

```
[사용자 브라우저]
       ↓ (URL 접속)
[프론트엔드 - React]  ←→  [백엔드 - Express]  ←→  [MySQL DB]
   (화면 그리기)           (데이터 처리)            (데이터 저장)
       ↓                       ↓
   Vercel 배포              Render 배포(?)
                               ↓
                        [Firebase Storage]
                          (이미지 저장)
```

- **프론트엔드**: 사용자가 보는 화면. React로 만듦. `src/` 폴더
- **백엔드**: 프론트에서 "데이터 줘!" 하면 DB에서 꺼내서 보내줌. `server/` 폴더
- **DB**: 실제 데이터가 저장된 곳. MySQL (Aiven 클라우드)
- **Firebase**: 이미지 파일 저장소

## 2. 프론트엔드 구조 (`src/`)

### 프로젝트 루트 파일
| 파일 | 역할 |
|---|---|
| `index.html` | HTML 진입점 - Vite가 이 파일을 기반으로 앱을 빌드 |
| `vite.config.js` | Vite 빌드 설정 |
| `eslint.config.js` | ESLint 코드 규칙 설정 |
| `vercel.json` | Vercel 배포 설정 |
| `package.json` | 프론트엔드 의존성 및 스크립트 |
| `public/` | 정적 파일 (favicon.svg, icons.svg, MarkerClustering.js) |

### 핵심 파일
| 파일 | 역할 |
|---|---|
| `main.jsx` | **시작점** - 앱을 화면에 그려주는 첫 번째 파일. Redux Provider도 여기서 감쌈 |

### 데이터 흐름 (예: 게시글 목록 보기)

```
1. 사용자가 /community 접속
2. App.jsx가 URL 보고 → Community.jsx 렌더링
3. Community.jsx가 로딩되면서 → apiAxios.js의 getBoardList() 호출
4. getBoardList()가 → 백엔드 서버에 GET /api/board 요청
5. 백엔드가 → DB에서 게시글 꺼내서 → JSON으로 응답
6. Community.jsx가 → 받은 데이터로 화면 그리기
```

## 3. 백엔드 구조 (`server/`)

### 핵심 파일
| 파일 | 역할 |
|---|---|
| `index.js` | **서버 시작점** - Express 서버 생성, 라우터 연결, 포트 8080에서 실행 |
| `db.js` | **DB 연결** - MySQL 연결 풀 설정. 모든 라우터가 이 파일을 가져다 씀 |

| `missing_tables.sql` | 누락 테이블 생성 SQL |

## 4. DB 테이블 관계

```
USER (사용자)
 ├── BOARD (게시글) ── BOARD_COMMENT (댓글)
 │                  └── BOARD_LIKE (좋아요)
```

## 6. 개발할 때 알아야 할 것

### 새 페이지 추가하는 법
1. `src/pages/` 에 폴더 만들기 (예: `NewPage/NewPage.jsx`)
2. `App.jsx`에 Route 추가: `<Route path="/new" element={<NewPage />} />`
3. 필요하면 `apiAxios.js`에 API 함수 추가

### 새 API 추가하는 법
1. `server/routes/`에서 해당 라우터 파일 열기
2. `router.get()` 또는 `router.post()` 추가
3. DB 쿼리 작성 (`const [rows] = await pool.query(...)`)
4. 프론트 `apiAxios.js`에 호출 함수 추가

### 로그인 유저 정보 사용하는 법
```jsx
import { useSelector } from 'react-redux';

function MyComponent() {
  const { user, isLoggedIn } = useSelector((state) => state.user);
  // user.userNum, user.nickname 등 사용 가능
}
```
